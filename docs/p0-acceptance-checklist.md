# P0 Mojito 原型验收清单

> 适用版本：当前 `main` 的 P0 原型实现  
> 目标：确认“可理解、可完成、可反馈”的核心调酒体验已经稳定

---

## 1. 核心流程验收（必须通过）

- [ ] 从酒吧首页点击「开始营业」可进入调酒台
- [ ] 订单卡片能显示 Mojito 与客人偏好（清爽 / 薄荷香 / 不太甜）
- [ ] 可选择 5 种原料：白朗姆、青柠汁、糖浆、苏打水、薄荷
- [ ] 可执行 4 个核心动作：倒入/加料、加冰、摇酒、出杯
- [ ] 点击出杯后进入评分页，显示分数、评级、反馈条目
- [ ] 可从评分页执行「再调一杯」和「回到酒吧」

---

## 2. 交互边界验收（P0 收口重点）

- [ ] 长按倒料后松手，倒料动画与状态能正确停止
- [ ] 倒料中切换页面（出杯或重置）不会出现残留倒料任务
- [ ] 量杯有剩余液体时点击出杯，液体会自动计入本杯后再评分
- [ ] 返回酒吧时不会出现“先跳回调酒台再切回首页”的闪动
- [ ] 连续快速点击加冰/摇酒/出杯，不会导致页面卡死或状态异常

---

## 3. 配方与评分验收（建议基准）

### 推荐基准配方（目标）
- 白朗姆：45ml
- 青柠汁：15ml
- 糖浆：15ml
- 苏打水：90ml
- 薄荷：8 片
- 冰块：约 6 块
- 摇酒：至少 3 次

### 评分反馈检查
- [ ] 缺少某原料时，会出现对应“缺少 xxx”反馈
- [ ] 冰块过少/过多时，会出现温度或稀释相关反馈
- [ ] 摇酒不足时，会出现融合度不足反馈
- [ ] 接近目标用量时，能看到正向反馈

---

## 4. 演示通过标准（最小可发布到内部测试）

- [ ] 新用户在无讲解情况下可完成一杯 Mojito
- [ ] 演示 3 轮（标准配方 / 明显偏差 / 随机操作）均可得到合理反馈
- [ ] 无阻断性问题（崩溃、无法出杯、无法重置、页面卡死）
