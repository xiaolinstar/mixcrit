# mixcrit P0.8 迭代计划

> 制定日期：2026-06-14  
> 更新日期：2026-06-15  
> 基线版本：0.0.8 (8) TestFlight 已发布版
> 目标版本：0.0.9

---

## 1. 迭代定位

P0.8 是 `0.0.8 (8)` TestFlight 基线之后的体验修正版本，不扩展新酒单，也不引入经营系统。

当前 TestFlight 外部反馈较少，因此 P0.8 不等待外部反馈再启动。此阶段采用“内部体验验收驱动”：先修正已经明确暴露的调酒页理解、布局和视觉层级问题，把单杯 Mojito 体验打磨到值得扩大测试的状态。

版本结论：

| 项目 | 值 |
|------|----|
| 当前已发布 TestFlight | `0.0.8 (8)` |
| 当前已发布产品阶段 | `P0.7` |
| 下一个版本 | `P0.8 / 0.0.9 (9+)` |
| 版本主题 | 调酒页理解、控件 UI 与杯具关系修正 |
| 不进入本版 | 第二杯酒、经营系统、登录、存档 |

这一版聚焦四件事：

- 新用户能否在 3 分钟内理解并完成第一杯 Mojito
- 调酒台视觉层级和操作关系是否足够清楚
- 真机尺寸、触控、动画和性能是否稳定
- 是否达到可以继续扩大 TestFlight 测试范围的质量

## 2. 优先级

- [ ] 优化量杯、成品杯、倒入方向和吧台板的视觉关系
- [ ] 检查并修复小屏设备上的文字拥挤、按钮误触、布局重叠
- [ ] 更新 TestFlight Beta 描述和测试说明
- [ ] 将版本号提升到 0.0.9，build 号提升到 9

### P1：高价值优化

- [ ] 调酒台增加更明确的当前步骤提示
- [ ] 强化出杯前的状态确认，减少误出杯
- [ ] 调整评分文案，让玩家知道下一杯如何改进
- [ ] 对首屏首页和调酒台做真机截图记录
- [ ] 补充最小反馈入口或反馈指引

### P2：暂缓

- [ ] 酒吧经营系统

## 3. 输入来源

P0.8 不以 TestFlight 反馈作为启动前置条件。输入来源按优先级排序：

| 来源 | 用途 | 状态 |
|------|------|------|
| 内部目视验收 | 发现布局、层级、视觉理解问题 | 主要输入 |
| 模拟器/真机回归 | 发现尺寸、安全区、触控问题 | 主要输入 |
| TestFlight 反馈 | 捕捉真实用户无法预测的问题 | 辅助输入 |
| 代码结构审查 | 判断 SpriteKit/SwiftUI 边界是否清晰 | 辅助输入 |

如收到外部测试反馈，继续按以下标签整理：

| 标签 | 判断标准 |
|------|----------|
| `onboarding` | 不知道下一步做什么、看不懂玩法、第一次失败 |
| `interaction` | 按住、松手、量杯入杯、摇酒、出杯等操作不顺 |
| `visual` | 原料、量杯、杯子、动画、层级、审美问题 |
| `layout` | 小屏、大屏、横竖屏、安全区、文字截断 |
| `scoring` | 分数不理解、反馈不公平、目标不清楚 |
| `stability` | 崩溃、卡死、明显掉帧、资源缺失 |

## 4. 验收标准

- [ ] 至少 3 名外部测试者完成一次 Mojito 流程
- [ ] 没有阻塞型崩溃或无法出杯的问题
- [ ] 首次进入调酒台后，测试者能理解原料、量杯、玻璃杯的关系
- [ ] 小屏设备没有关键按钮或文字不可见
- [ ] 调酒页在 iPhone SE、iPhone 17、Pro Max 级别尺寸上保持单屏完成核心动作
- [ ] 原料区视觉权重高于动作按钮，动作按钮不再抢占主要注意力
- [ ] TestFlight 说明与当前版本功能一致
- [ ] Xcode Archive 成功并上传 0.0.9 (9)

---

## 5. 建议执行顺序

1. 收口 `ingredient-priority-layout`：确认原料区和按钮主次关系
2. 执行 `jigger-glass-relationship`：让量杯、成品杯、倒入方向更直观
3. 回归小屏布局：确认没有遮挡、挤出、安全区问题
4. 更新 TestFlight Beta 描述，明确本版测试重点
5. 将 Xcode 版本提升到 `0.0.9 (9)`
6. Archive 并上传下一版 TestFlight

---

## 5.0 执行原则

- 先修确定性问题，不等待外部反馈。
- 每个 P0 改动必须对应已有 Spec 或先补 Spec。
- 每轮 UI 改动后至少保存一张 iPhone 17 级别截图。
- 发布前必须补 iPhone SE 与 Pro Max 级别尺寸回归。
- 只有当 Mojito 单杯流程清楚、稳定、单屏可完成时，才扩大 TestFlight 测试范围。

## 5.2 当前执行批次

### Batch A：调酒页视觉主次

目标：让玩家一眼知道底部原料是主要操作对象，动作按钮只是辅助工具。

状态：In Progress。

验收：

- [x] 原料区视觉权重高于动作按钮。
- [x] 动作按钮固定高度，不再因系统按钮样式膨胀。
- [ ] 小屏设备确认不截断、不遮挡、不挤出。

### Batch B：杯具关系与舞台语义

目标：让玩家知道左侧是量杯，右侧是成品杯，并理解“量杯入杯”是左到右的转移动作。

状态：Next。

验收：

- [ ] 舞台中有明确但不喧宾夺主的左到右关系提示。
- [ ] 橙黄色/棕色吧台板不再像一个无意义的大块按钮。
- [ ] 空成品杯的玻璃质感更精致。
- [ ] 加冰、薄荷、苏打后的杯内反馈明显。

### Batch C：发布回归

目标：把 P0.8 变成可上传的 `0.0.9 (9)`。

状态：Pending。

验收：

- [ ] iPhone SE、iPhone 17、Pro Max 级别尺寸截图通过。
- [ ] 首页、调酒页、评分页主路径通过。
- [ ] TestFlight Beta 描述更新。
- [ ] Xcode 版本号提升到 `0.0.9 (9)`。
