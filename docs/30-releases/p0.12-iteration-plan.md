# mixcrit P0.12 迭代计划

> 制定日期：2026-06-25  
> 基线版本：`0.0.13 (13+)` — P0.11 调酒主舞台打磨  
> 目标版本：`0.0.14 (14+)`  
> 主题：简化调酒循环验证  
> 当前状态：发布候选（方向试验版）

---

## 1. 迭代定位

P0.12 是一次产品方向验证，不是单纯功能扩展。

在确认 mixcrit 未来是养成类游戏后，P0.12 要验证调酒流程是否可以从完整真实模拟，转为更适合经营循环的短时高频制作小玩法。

本版本暂不新增第二杯酒，也不做金币经验和完整经营系统。先用 Mojito 验证新的单杯形态。

---

## 2. 目标

- 将一杯 Mojito 的常规完成时间压缩到 20-40 秒。
- 移除或弱化重复量杯点击。
- 引入顾客口味偏好。
- 在简化操作中保留调酒知识：用量、风味影响和手法原因需要可见。
- 形成新的出杯反馈结构：配方、偏好、手法、呈现、速度。
- 为 P0.13 配方系统化和 P1.0 经营养成打基础。

---

## 3. 不进本版

- 第二杯酒。
- 库存、金币、经验、每日营业。
- 大规模 SpriteKit 视觉重构。
- 第三方动画库。
- 完整商业化或付费设计。

---

## 4. 任务清单

### Phase A — 玩法原型

- [x] 定义 `CustomerPreference` 和 4 种 Mojito 偏好。
- [x] 定义简化 Mojito 状态：酒感、甜度、薄荷、冰量、手法。
- [x] 设计简化流程入口，可通过 Debug 或内部开关进入。
- [x] 用当前 UI 快速搭出简化调酒流程。

### Phase B — 出杯反馈

- [x] 新增出杯反馈评价器：内部计算分数，外部输出顾客反馈。
- [x] 出杯反馈页显示顾客反应、表现标签和收益变化。
- [x] 从“ml 偏差”反馈改为“口味解释”反馈。
- [x] 将“轻 / 标准 / 重”改为真实用量和风味说明。
- [ ] 记录完成时间，用于速度评价。

### Phase C — 手感与视觉复用

- [ ] 保留一次倒酒目标区动作。
- [ ] 保留一次搅拌/摇酒节奏动作。
- [x] 保留出杯反馈和杯内状态变化。
- [ ] 减少量杯在默认流程中的强制存在感。

### Phase D — 验收

- [x] iPhone 17e 模拟器完成 3 轮流程截图验收。
- [x] 对比 P0.11：默认操作次数明显下降，从多次量杯装填转为档位选择 + 3 次摇拌 + 出杯。
- [x] 更新产品决策：继续简化，旧写实流程保留为参考/专家模式候选。
- [x] `xcodebuild` Debug 模拟器构建通过。

---

## 5. 关键决策点

P0.12 结束时必须回答：

1. 简化流程是否仍然有调酒感？
2. 玩家是否更愿意连续完成多杯？
3. 客人口味偏好是否带来有效决策？
4. 量杯是否只保留在教学/专家模式？
5. P0.13 是否可以进入配方系统化？
