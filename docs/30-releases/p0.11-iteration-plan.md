## 1. 迭代定位

P0.11 是 **调酒主玩法定稿版**。

它不是养成系统版本，而是进入养成系统之前的最后一轮核心玩法打磨：让 Mojito 调酒页在视觉、反馈、构图和小屏适配上足够稳定，后续 P1 才能把顾客、金币、经验、解锁等养成层建立在可靠的调酒体验上。

> **2026-05-26 决策**：构图与基础反馈已落地并真机验收（UI 优于 0.0.12），但整体质感与动作重量感仍不足。**继续在 P0.11 / 0.0.13 范围内打磨调酒 UI 与动画，暂不进入 P0.12 扩配方**。打磨项清单与验收标准见 `docs/20-specs/p0.11/mixing-visual-animation-polish.md`。

## 2. 为什么不是直接 P1.0

mixcrit 未来要做成养成类游戏，但 P1 的养成闭环依赖一个可复用的“调一杯酒”核心。

如果当前 Mojito 调酒页仍存在主舞台空洞、动作反馈弱、玩家不确定点击后发生了什么，那么提前加入经营系统会带来三个风险：

- 养成奖励无法掩盖核心操作不爽的问题。
- 后续每新增一杯酒都会重复修 UI 和动画债务。
- 经济循环参数缺少稳定基线，例如一杯酒耗时、失败点、评分体验。

因此 P0.11 先把调酒体验打磨到可作为视觉与反馈基线；P0.12 转向验证更适合养成游戏的简化调酒循环；P0.13 再处理配方扩展，P1.0 才进入养成 MVP。
