# mixcrit P0.8 迭代计划

> 制定日期：2026-06-14
> 基线版本：0.0.8 (8) TestFlight 候选版
> 目标版本：0.0.9

---

## 1. 迭代定位

P0.8 是 TestFlight 首轮反馈后的体验修正版本，不扩展新酒单，也不引入经营系统。

这一版聚焦四件事：

- 新用户能否在 3 分钟内理解并完成第一杯 Mojito
- 调酒台视觉层级和操作关系是否足够清楚
- 真机尺寸、触控、动画和性能是否稳定
- 是否形成可持续收集外部测试反馈的闭环

---

## 2. 优先级

### P0：必须完成

- [ ] 收集首批 TestFlight 反馈，按问题类型归类
- [ ] 修正影响首次完成 Mojito 的操作理解问题
- [ ] 检查并修复小屏设备上的文字拥挤、按钮误触、布局重叠
- [ ] 补充最小反馈入口或反馈指引
- [ ] 更新 TestFlight Beta 描述和测试说明
- [ ] 将版本号提升到 0.0.9，build 号提升到 9

### P1：高价值优化

- [ ] 调酒台增加更明确的当前步骤提示
- [ ] 优化量杯到玻璃杯的关系表达
- [ ] 强化出杯前的状态确认，减少误出杯
- [ ] 调整评分文案，让玩家知道下一杯如何改进
- [ ] 对首屏首页和调酒台做真机截图记录

### P2：暂缓

- [ ] 新增第二杯鸡尾酒
- [ ] 酒吧经营系统
- [ ] 库存、货币、升级系统
- [ ] 音效完整系统
- [ ] Game Center、登录、云存档

---

## 3. 反馈归类

首轮外部测试反馈按以下标签整理：

| 标签 | 判断标准 |
|------|----------|
| `onboarding` | 不知道下一步做什么、看不懂玩法、第一次失败 |
| `interaction` | 按住、松手、量杯入杯、摇酒、出杯等操作不顺 |
| `visual` | 原料、量杯、杯子、动画、层级、审美问题 |
| `layout` | 小屏、大屏、横竖屏、安全区、文字截断 |
| `scoring` | 分数不理解、反馈不公平、目标不清楚 |
| `stability` | 崩溃、卡死、明显掉帧、资源缺失 |

---

## 4. 验收标准

- [ ] 至少 3 名外部测试者完成一次 Mojito 流程
- [ ] 没有阻塞型崩溃或无法出杯的问题
- [ ] 首次进入调酒台后，测试者能理解原料、量杯、玻璃杯的关系
- [ ] 小屏设备没有关键按钮或文字不可见
- [ ] TestFlight 说明与当前版本功能一致
- [ ] Xcode Archive 成功并上传 0.0.9 (9)

---

## 5. 建议执行顺序

1. 等待首批 TestFlight 外部测试反馈
2. 汇总阻塞问题和高频困惑点
3. 先修 P0 问题，再做 P1 体验优化
4. 真机验证首页、调酒台、评分页三条主路径
5. 提交 0.0.9，并上传下一版 TestFlight
