# P0: Mojito 调酒台原型设计

日期：2026-05-19  
项目：mixcrit  
版本目标：确认第一个可玩版本  
版本名称：P0: Mojito 调酒台原型

## 1. P0 目标

P0 的目标不是做完整游戏，而是验证核心体验：

> 玩家在手机上调一杯 Mojito，是否有趣、好看、可理解？

P0 要回答四个问题：

1. 倒酒、加冰、摇酒、出杯这些动作是否顺手？
2. 调酒台的信息布局是否清晰？
3. 评分反馈是否能让玩家理解自己做得好不好？
4. 视觉方向是否能表达“这杯酒看起来好喝”？

## 2. P0 范围

### 2.1 本版本做什么

- 一个酒吧入口界面
- 一个 Mojito 客人订单
- 一个调酒台界面
- Mojito 所需 5 种材料
- 倒酒、加冰、摇酒、出杯 4 个核心动作
- 简单步骤检查
- 简单用量评分
- 简单成品视觉反馈

### 2.2 本版本不做什么

- 不做完整经营系统
- 不做多客人队列
- 不做进货和库存扣减
- 不做真实营养数据库
- 不做复杂液体物理
- 不做完整角色动画
- 不做商店、内购、账号和云同步

## 3. P0 用户流程

1. 用户进入酒吧首页
2. 点击“开始营业”
3. 客人点单：Mojito
4. 用户进入调酒台
5. 用户按提示加入材料
6. 用户加冰
7. 用户摇酒
8. 用户出杯
9. 系统展示评分和反馈
10. 用户可以重新调制

## 4. Mojito 配方数据

P0 使用简化配方，方便快速验证。

| 材料 | 目标用量 | 单位 | 类型 |
| --- | ---: | --- | --- |
| 白朗姆 | 45 | ml | 基酒 |
| 青柠汁 | 20 | ml | 果汁 |
| 糖浆 | 15 | ml | 糖浆 |
| 苏打水 | 90 | ml | 碳酸饮料 |
| 薄荷 | 8 | 片 | 装饰 / 香草 |
| 冰块 | 6 | 颗 | 冰 |

P0 允许误差：

- 液体材料误差在 5ml 内为优秀
- 误差在 10ml 内为可接受
- 超过 10ml 会明显扣分

## 5. 调酒台布局

### 5.1 顶部区域

展示：

- 当前订单：Mojito
- 客人提示：清爽、薄荷香、不要太甜
- 当前步骤提示

### 5.2 中央区域

展示：

- 主杯具
- 液面高度
- 液体颜色
- 冰块数量
- 摇酒状态
- 成品状态

这是最重要的视觉区域，后续会迁移到 SpriteKit 或更细的动画层。

### 5.3 底部区域

展示材料按钮：

- 白朗姆
- 青柠汁
- 糖浆
- 苏打水
- 薄荷
- 冰块

### 5.4 操作区域

展示核心动作：

- 加入选中材料
- 加冰
- 摇酒
- 出杯
- 重置

## 6. 交互规则

### 6.1 倒酒

P0 采用按钮式倒酒，不直接做倾斜手机。

规则：

- 点击材料按钮选中材料
- 点击“加入”会按固定步进加入材料
- 不同材料步进不同
  - 液体：每次 5ml
  - 薄荷：每次 2 片
- 用户可以多次点击，模拟控制用量

后续升级：

- 长按持续倒酒
- 根据按压时长控制毫升数
- 根据手机倾斜角控制流速

### 6.2 加冰

P0 使用按钮加冰。

规则：

- 每次点击加 1 颗冰
- 冰块数量影响评分
- 冰块数量会在杯中以小方块视觉呈现

后续升级：

- 拖动冰块到杯中
- 冰块落杯碰撞动画
- 冰块影响温度和稀释度

### 6.3 摇酒

P0 使用点击按钮触发摇酒。

规则：

- 每次点击“摇酒”增加一次摇酒强度
- 摇酒次数达到 3 次为合格
- 摇酒不足会扣分

后续升级：

- 使用 Core Motion 检测真实摇动
- 统计摇酒强度、节奏和时长
- 加入 Core Haptics 反馈

### 6.4 出杯

规则：

- 点击“出杯”进入结算
- 系统根据材料、冰块、摇酒和步骤完成度计算评分

## 7. 评分规则

总分 100 分。

评分维度：

- 用量准确度：60 分
- 冰块数量：10 分
- 摇酒完成度：15 分
- 材料完整度：10 分
- 操作完成度：5 分

评级：

- S：90 到 100 分
- A：80 到 89 分
- B：70 到 79 分
- C：60 到 69 分
- D：60 分以下

## 8. P0 动画设计

P0 的动画目标是“足够让人相信这是调酒”，不用追求完整物理真实。

### 8.1 必做动画

- 点击加入材料时，杯中液面上升
- 液体颜色随材料加入发生变化
- 冰块数量增加时，杯中出现冰块
- 摇酒时杯具或摇壶出现抖动
- 出杯后展示成品卡片和评分跳出

### 8.2 后续动画升级

- 酒液从瓶口流出
- 冰块落杯和反弹
- 摇壶金属高光
- 杯壁冷凝水
- 薄荷叶轻微摆动
- 苏打气泡上升

## 9. P0 图片和视觉资产

P0 可以先使用 SwiftUI 绘制占位视觉，不依赖外部图片。

需要表达的视觉对象：

- 酒吧背景
- Mojito 成品杯
- 材料按钮
- 冰块
- 薄荷
- 客人订单卡
- 评分卡

P0 视觉原则：

- 深色酒吧背景
- 暖色灯光
- Mojito 使用清透浅绿
- 杯子和液体要成为视觉中心
- UI 不遮挡杯子

## 10. 技术实现建议

P0 使用 SwiftUI 单文件快速实现。

技术点：

- `ContentView` 维护状态
- 使用 `enum` 管理界面阶段
- 使用 `struct` 定义材料和评分
- 使用 SwiftUI Shape 绘制杯子、液面和冰块
- 使用动画修饰器实现液面上升和摇动

P1 再考虑拆分：

- `GameState`
- `Recipe`
- `Ingredient`
- `BarView`
- `MixingStationView`
- `ScoreView`
- SpriteKit 调酒台场景

## 11. P0 验收标准

P0 完成后应满足：

- App 启动后能看到酒吧入口
- 点击开始营业进入 Mojito 订单
- 用户能加入材料、加冰、摇酒、出杯
- 出杯后能看到总分、评级和具体反馈
- 用户可以重置并重新调制
- 画面具备初步酒吧和 Mojito 氛围

## 12. 第一个版本确认

第一个版本确认如下：

> P0: Mojito 调酒台原型

本版本只追求一个小闭环：

> 开始营业 -> Mojito 订单 -> 调酒台操作 -> 出杯评分 -> 重新调制

只要这个闭环跑通，就可以进入 P1：小酒吧经营 MVP。
