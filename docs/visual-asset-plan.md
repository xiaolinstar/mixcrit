# P0.4 视觉资产与调酒台交互规划

日期：2026-05-21  
阶段：P0.4  
建议版本号：0.0.5

## 1. 当前状态

当前 App 没有真实贴图，调酒台、酒瓶、杯子、量杯、冰块、薄荷和气泡都由 SwiftUI 的 `Shape`、渐变、描边和系统图标绘制。

这些内容适合交互验证，但不足以支撑最终产品的视觉表现。后续需要逐步替换为统一风格的 PNG 或矢量资源。

## 2. P0.4 目标

P0.4 的目标是把调酒台从“按钮表单”调整为“吧台物件操作”：

- 材料从按钮网格改为吧台陈列
- 用户按住酒瓶或材料物件开始倒入
- 冰桶作为吧台物件存在
- 量杯和成品杯保持视觉中心
- 底部只保留工具动作，例如量杯入杯、摇酒、出杯

## 3. 视觉风格方向

建议风格：

- 半写实
- 温暖酒吧灯光
- 玻璃和液体质感突出
- UI 低调，不抢调酒物件
- 角色和经营系统后续可轻度风格化

避免：

- 过度卡通
- 过度写实导致资产成本过高
- 大面积纯按钮界面
- 用系统图标替代关键调酒物件

## 5. 后续替换策略

当前代码中的 SwiftUI 占位物件应逐步替换为图片：

- `BarIngredientObjectView` -> 酒瓶和薄荷贴图
- `IceBucketObjectView` -> 冰桶贴图
- `JiggerView` -> 量杯贴图 + SwiftUI 刻度层
- `CocktailGlassView` -> 杯具贴图 + SwiftUI 液体层

推荐不要一次性替换全部。优先替换：

1. 成品杯
2. 量杯
3. 白朗姆瓶
4. 冰块
5. Mojito 剩余材料
6. 吧台背景

## 6. 交互优化方向

当前 P0.4 交互规则：

- 按住酒瓶倒入量杯
- 按住苏打水直接倒入杯中
- 按住薄荷按片加入杯中
- 点击冰桶加冰
- 点击量杯入杯
- 点击摇酒和出杯

后续优化：

- 拖动量杯到杯子上方倒入
- 拖动冰块到杯中
- 拖动薄荷到杯中
- 长按酒瓶时显示瓶身倾斜动画
- 倒入时显示更真实的液体流

## 7. P0.4 验收标准

- 调酒台看起来像吧台，而不是表单
- 材料以物件形式陈列
- 用户可以通过按住物件完成核心操作
- 量杯和成品杯仍然是视觉中心
- 代码结构方便后续替换为贴图资源
