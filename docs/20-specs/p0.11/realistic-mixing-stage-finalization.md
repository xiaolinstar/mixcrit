# P0.11 写实调酒主舞台定稿 Spec

> 创建日期：2026-06-18  
> 产品阶段：P0.11  
> 目标发布标识：0.0.13 (13+)  
> 状态：Draft  
> 前置基线：P0.10 / 0.0.12 首次体验收口

---

## 1. 背景

mixcrit 的长期方向是养成类调酒游戏，但养成循环必须建立在稳定、好懂、好玩的单杯调酒核心上。

P0.9 已完成写实资产方向与 Liquid Glass 风格探索，P0.10 已补充动态步骤提示和首次引导。当前剩余主要问题不再是“玩家是否知道下一步”，而是“主舞台是否足够像真实调酒、动作是否足够有反馈”。

当前调酒页仍存在：

- 主舞台上方暗区偏大，视觉重心不够接近参考 UI 图。
- 量杯、成品杯虽然更清楚，但还没有形成足够强的真实摄影感。
- 点击原料后，量杯/成品杯反馈不够强，玩家不能立刻感知“我刚刚做了什么”。
- 加冰、摇酒、量杯入杯等动作仍偏 UI 触发，缺少游戏感。

P0.11 的目标是把 Mojito 调酒主玩法打磨到可作为后续养成系统的稳定核心。

---

## 2. 目标

- 调酒主舞台接近参考图的构图：大杯具、少空洞、强聚焦。
- 量杯和成品杯成为屏幕主角，而不是 UI 面板中的两个元素。
- 原料点击、量杯入杯、加冰、摇酒、出杯前状态都有清晰反馈。
- 底部原料仍保持足够大，但所有小屏机型不溢出。
- 形成后续可复用到更多酒品的动画与舞台布局规则。

---

## 3. 非目标

- 不新增第二杯酒。
- 不引入经营、金币、经验、顾客、装修系统。
- 不做真实流体物理模拟。
- 不重写整体 App 架构。
- 不把 P1 养成系统提前塞进 P0.11。

---

## 4. 用户体验

玩家进入调酒页后，应首先看到中央调酒主舞台：

1. 左侧写实量杯，右侧写实成品杯，二者占据主要屏幕面积。
2. 顶部 HUD 只承担订单和下一步提示，不压迫主舞台。
3. 底部原料像真实吧台陈列，图片足够大，文字只做辅助。
4. 当前推荐原料或动作有轻微发光/抬起反馈。
5. 点击量杯型原料后，量杯立刻出现液体、轻微晃动、数值变化。
6. 点击「量杯入杯」后，出现从量杯到成品杯的倒入路径，成品杯液面/内容随之变化。
7. 点击「加冰」后，冰块从上方或侧上方落入杯中。
8. 点击「摇酒」后，成品杯或摇酒状态有明显节奏动画，不只是数字变化。

## 5. 设计方案

### 5.1 页面层级

从上到下固定为：

| 区域 | 作用 | 约束 |
|------|------|------|
| 顶部 HUD | 订单、当前目标、下一步 | 胶囊高度受控，不与主舞台重叠 |
| 主舞台 | 量杯、成品杯、倒入/加冰/摇酒反馈 | 屏幕视觉主角，优先分配高度 |
| 原料托盘 | 白朗姆、青柠汁、糖浆、苏打水、薄荷 | 贴近底部，图片优先，文字收敛 |
| 动作栏 | 量杯入杯、加冰、摇酒、出杯、清空 | 小而清楚，不抢原料权重 |

### 5.2 主舞台构图

- 量杯与成品杯整体上移，减少主舞台顶部空洞。
- 成品杯比量杯更高、更靠右，保持参考图的视觉节奏。
- 量杯和成品杯不显示多余文字标签。
- 台面阴影和杯底反光只服务于真实感，不做装饰性大背景。
- 倒入箭头仅在有液体需要转移时强化，平时弱化。

### 5.3 动画优先级

| 优先级 | 动画 | 验收重点 |
|--------|------|----------|
| P0 | 原料入量杯 | 点击后量杯液体可见，反馈即时 |
| P0 | 量杯入杯 | 路径清晰，成品杯内容变化明确 |
| P0 | 加冰 | 冰块落杯轨迹与杯内冰块增加一致 |
| P0 | 摇酒 | 有节奏晃动，完成后状态变化可感知 |
| P1 | 出杯前高光 | 可作为后续评分前仪式感 |

### 5.4 小屏适配

验收设备至少包括：

- iPhone SE 或最小可用模拟器
- iPhone 17e
- iPhone 17 / 15 Pro
- Pro Max 级别大屏

所有设备必须满足：

- 无横向溢出。
- 底部原料和动作栏均可见。
- 主舞台不被 HUD 或底部 Dock 遮挡。
- 核心动作单屏完成，不需要滚动。

## 6. 技术方案

### 6.1 主要涉及文件

| 文件 | 改动方向 |
|------|----------|
| `mixcrit/Game/MixingScene.swift` | 主舞台构图、杯具位置、倒入/加冰/摇酒动画 |
| `mixcrit/Game/MixingSceneLayout.swift` | 抽离不同屏幕尺寸下的量杯/成品杯布局规则 |
| `mixcrit/Views/MixingStation/MixingStationView.swift` | HUD、主舞台、底部 Dock 高度分配 |
| `mixcrit/Views/MixingStation/IngredientObjects.swift` | 原料卡片尺寸、推荐态、文字压缩 |
| `mixcrit/Views/Components/ActionButton.swift` | 动作按钮降权但保持可读 |
| `mixcrit/Assets.xcassets` | 如需要，继续替换杯具/冰块/液体资产 |

### 6.2 实现顺序

1. 固定 P0.11 目标截图基线：当前 iPhone 17e 常态截图作为 before。
2. 调整 SpriteKit 主舞台构图，减少上方暗区。
3. 放大量杯与成品杯，建立小屏尺寸保护规则。
4. 强化原料入量杯、量杯入杯、加冰、摇酒四类动画。
5. 回归底部原料托盘，不允许白朗姆/薄荷溢出。
6. 多机型截图验收。

## 7. 验收标准

- [ ] iPhone 17e 常态页中，主舞台上方空洞明显减少，量杯与成品杯成为视觉主角。
- [ ] iPhone SE / 最小屏模拟器中，底部白朗姆和薄荷不溢出屏幕。
- [ ] 点击白朗姆、青柠汁、糖浆后，量杯内液体反馈在 0.2 秒内可见。
- [ ] 点击「量杯入杯」后，成品杯出现明确液体/内容变化。
- [ ] 点击「加冰」后，至少有一次可见落冰动画并与冰块计数一致。
- [ ] 点击「摇酒」后，画面有明显节奏动画，且不会导致布局跳动。
- [ ] 顶部 HUD 不与主舞台边界重叠。
- [ ] 底部原料托盘贴近屏幕底部，无大面积无意义留白。
- [ ] `xcodebuild` Debug 模拟器构建通过。
- [ ] 至少输出 iPhone 17e 与 Pro Max 两张验收截图。

---

## 8. 与 P1 养成系统的关系

P0.11 不做养成系统，但为 P1 做基础：

- 调酒动作成为后续“订单表现”和“评分反馈”的稳定核心。
- 原料、动作、杯具的视觉规则可复用到更多酒品。
- 玩家完成一杯酒的操作时长、失败点、评分反馈将作为 P1 经济循环参数的输入。

P1.0 才开始引入：

- 顾客订单
- 金币 / 经验
- 酒单解锁
- 吧台升级
- 每日循环
