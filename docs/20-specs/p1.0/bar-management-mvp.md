# P1.0 酒吧养成 MVP Spec

> 创建日期：2026-06-18  
> 产品阶段：P1.0  
> 目标发布标识：0.1.0  
> 状态：Draft  
> 前置条件：P0.11 调酒主舞台定稿、P0.12 配方系统化完成

---

## 1. 背景

mixcrit 的长期方向是 **真实调酒模拟 + 轻经营养成**。

P0 阶段验证单杯 Mojito 的调酒操作是否成立；P1 阶段开始把调酒小游戏包进酒吧经营循环，让玩家有长期目标。

P1.0 不追求完整商业化和大量内容，而是验证最小养成闭环：

```text
顾客下单 → 玩家调酒 → 系统评分 → 获得金币/经验
→ 解锁/升级 → 进入下一单或下一天
```

---

## 2. 目标

- 让玩家完成一个最小营业日。
- 把调酒评分转化为金币、经验和顾客满意度。
- 提供最小成长目标：等级、酒单解锁、吧台升级。
- 保持调酒操作仍是核心，不让经营 UI 抢走产品灵魂。

---

## 3. 非目标

- 不做复杂剧情。
- 不做多人、排行榜、社交。
- 不做完整库存经济平衡。
- 不做 10 款以上酒品。
- 不做付费系统。

---

## 4. MVP 核心循环

1. 玩家进入酒吧首页。
2. 点击开始营业。
3. 系统生成 1 个顾客订单。
4. 玩家进入调酒台完成订单。
5. 系统根据调酒评分生成金币、经验、满意度。
6. 返回酒吧首页，显示收益与成长。
7. 达成条件后解锁新配方或吧台升级。

---

## 5. P1.0 最小范围

| 模块 | P1.0 范围 |
|------|-----------|
| 顾客 | 1-3 个基础顾客模板 |
| 订单 | 单杯订单，复用 P0 调酒台 |
| 收益 | 金币、小费、经验 |
| 成长 | 玩家等级、酒单解锁 |
| 酒单 | Mojito + 1-2 款新增经典鸡尾酒 |
| 吧台 | 最小升级入口，先只影响视觉或收益倍率 |
| 结算 | 单杯结算 + 当日简报 |

---

## 6. 进入 P1.0 的条件

- [ ] P0.11 主舞台构图和动作反馈验收通过。
- [ ] P0.12 配方系统支持至少 2 款酒。
- [ ] 单杯调酒流程在无讲解情况下可完成。
- [ ] 评分反馈能被玩家理解。
- [ ] 至少一轮外部 TestFlight 反馈显示玩家愿意再调第二杯。

---

## 7. 验收标准

- [ ] 玩家可以完成一个完整营业日。
- [ ] 至少 1 个顾客订单能进入调酒台并返回结算。
- [ ] 调酒评分能转化为金币和经验。
- [ ] 达成条件后能看到明确成长反馈。
- [ ] P1 经营层不破坏 P0 调酒台体验。
- [ ] `xcodebuild` Debug 模拟器构建通过。
