# Spec: P0.8 量杯与成品杯关系表达

> 创建日期：2026-06-14  
> 更新日期：2026-06-15  
> 阶段：P0.8  
> 目标版本：0.0.9  
> 状态：In Progress

---

## 1. 背景

当前调酒舞台中同时出现左侧量杯和右侧透明玻璃杯，但用户可能不理解二者关系。同时，现有实现把量杯表达成“带刻度的连续计量容器”，与真实调酒中的 jigger 使用方式不一致。

- 左侧量杯用于白朗姆、青柠汁、糖浆的计量。
- 右侧玻璃杯是最终成品杯。
- “量杯入杯”动作会把左侧量杯中的液体倒入右侧成品杯。
- 真实 jigger 更接近固定容量单位，例如 15ml；需要 30ml 时应使用两次，而不是缓慢倒到任意刻度。

如果视觉没有明确表达，用户会疑惑：

- 为什么有两个杯子？
- 为什么同时有一个物理量杯和一个刻度量杯？
- 为什么可以缓慢倒到非标准容量？
- 当前到底应该往哪个杯子里倒？

---

## 2. 目标

- 明确左侧是“量杯”，右侧是“成品杯”。
- 明确量杯液体会通过“量杯入杯”进入成品杯。
- 移除无意义吧台板和说明文字，舞台只保留必要杯具与液体反馈。
- 后续将量杯交互调整为固定容量单位，不支持非刻度容量。
- 提升成品杯精致度，让最终饮品看起来更有吸引力。

---

## 3. 非目标

- 不做真实 3D 杯具。
- 不引入复杂液体物理。
- 不改变 Mojito 配方。
- 本批先处理舞台视觉；固定 15ml 单位交互作为下一批实现。

---

## 4. 设计方向

### 4.1 舞台关系

建议将舞台语义固定为：

```text
左：物理量杯 / Jigger
右：成品杯 / Glass
```

可以通过以下方式强化关系：

- 不在舞台上显示“量杯 / 成品杯”文字。
- 不显示额外外框刻度量杯。
- 只保留一个物理 jigger 贴图。
- 量杯到成品杯之间保留细微倒入轨迹占位。
- 成品杯显示液位、冰块、薄荷、气泡。
- 移除橙黄色/棕色吧台板。

### 4.2 成品杯精修方向

- 提高玻璃杯边缘高光。
- 依赖写实玻璃杯素材本身表达杯口和杯底，避免额外绘制上下两个圈。
- 液体、冰块、薄荷进入后增强层次。
- 避免空杯时过于像普通透明矩形。

### 4.3 量杯交互方向

当前连续倒酒与真实 jigger 认知不一致。下一批交互建议：

- 使用 15ml 作为基础量杯单位。
- 白朗姆 45ml = 15ml x 3。
- 青柠汁 15ml = 15ml x 1。
- 糖浆 15ml = 15ml x 1。
- 玩家每次对量杯执行一次“装量杯”，量杯获得一个固定单位。
- “量杯入杯”把当前量杯单位倒入成品杯。
- 不支持任意 17ml、22ml 这类非刻度容量。

---

## 5. 技术方案

主要修改：

- `mixcrit/Game/MixingScene.swift`
- `mixcrit/Game/MixingSceneLayout.swift`

### 5.1 实现拆解

第一步先做轻量 SpriteKit 精修，不引入新素材：

1. 移除吧台板
   - 不再渲染棕色/橙色台面。
   - 舞台只保留背景、量杯、成品杯、倒入轨迹和杯内反馈。

2. 加入杯具语义
   - 不通过文字标签解释。
   - 通过左侧物理 jigger 与右侧写实成品杯的差异解释。

3. 加入方向提示
   - 量杯与成品杯之间加入细线/箭头/虚线轨迹。
   - 当量杯中有液体时，提高该轨迹透明度。
   - 执行“量杯入杯”时，轨迹与液流方向一致。

4. 成品杯精修
   - 减少程序绘制的额外杯口/杯底圈。
   - 让写实玻璃杯贴图成为主视觉。
   - 空杯状态仍能看出玻璃体积。
   - 杯内内容不要被玻璃贴图完全盖住。

5. 小屏适配
   - 调整 `MixingSceneLayout` 中 jigger/glass 的 X 位置和比例。
   - 避免标签、箭头、杯具互相压住。

可选新增：

- SpriteKit node 命名常量
- 区域标签 node
- 更高质量的杯具资产

### 5.2 不在本批处理

- 不改底部原料 Dock。
- 不新增新酒或新动作。
- 不做复杂拖拽。
- 不做真实液体物理。

## 6. 验收标准

- [ ] 首次进入调酒页时，用户能从图形上判断左侧是量杯，右侧是成品杯。
- [ ] 用户能理解“量杯入杯”的方向是左到右。
- [ ] 舞台不再出现吧台板。
- [ ] 舞台不再显示“量杯 / 成品杯”文字标签。
- [ ] 舞台不再同时出现物理量杯和外层刻度量杯。
- [ ] 成品杯空杯状态仍具有玻璃质感。
- [ ] 加冰、加薄荷、加液体后成品杯视觉反馈明显。
- [ ] iPhone SE 级别小屏不出现标签、箭头、杯具互相遮挡。
- [ ] 后续交互改造中，标准酒类量杯容量按 15ml 单位处理，不支持非刻度容量。
