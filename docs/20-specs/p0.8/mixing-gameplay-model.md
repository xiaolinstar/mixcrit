# Spec: P0.8 调酒玩法模型重评估

> 创建日期：2026-06-15  
> 阶段：P0.8  
> 目标版本：0.0.9  
> 状态：Planned

---

## 1. 问题判断

当前调酒页的问题不是单纯 UI 混乱，而是玩法模型和真实调酒动作不一致。

现状：

- 白朗姆、青柠汁、糖浆：长按原料，连续倒入量杯，每次约 1.5ml。
- 苏打水：长按原料，连续直接倒入成品杯，每次约 6ml。
- 薄荷：长按原料，连续加入成品杯，每次 1 片。
- 冰块：点击按钮加入。
- 摇酒：点击按钮累计摇酒次数。
- 量杯：表现为一个连续刻度容器。

主要问题：

- 量杯不应该支持任意 17ml、22ml 这类非刻度容量。
- 15ml、30ml、45ml 这类容量应该通过固定 jigger 单位重复完成。
- Mojito 经典做法通常不需要雪克杯，当前“摇酒”动作和 Mojito 真实技法不完全匹配；但 P0.8 产品决策先保留 `摇酒` 文案，不显示雪克杯。
- 下方原料到底是点击、长按、倒入量杯、直接入杯，玩家无法一眼理解。
- 舞台杯具太小，视觉留白过多，核心对象没有成为主画面。

---

## 2. P0.8 玩法原则

P0.8 暂时只做好一杯 Mojito，不引入复杂酒谱系统。

原则：

- 手机操作以点击为主，避免长按连续计量。
- 标准酒类使用固定 15ml jigger 单位。
- 不支持非刻度容量。
- Mojito 不引入雪克杯。
- P0.8 先保留 `摇酒` 动作文案；真实技法上更接近 `搅拌/搅匀`，后续可在 P0.9 或 P1.0 统一改名。
- 舞台只保留当前玩法必要对象。

---

## 3. 建议 P0.8 Mojito 流程

### 3.1 固定单位

| 原料 | 目标 | 操作建议 |
|------|------|----------|
| 白朗姆 | 45ml | 点击白朗姆装入 15ml jigger，执行 3 次量杯入杯 |
| 青柠汁 | 15ml | 点击青柠汁装入 15ml jigger，执行 1 次量杯入杯 |
| 糖浆 | 15ml | 点击糖浆装入 15ml jigger，执行 1 次量杯入杯 |
| 苏打水 | 90ml | 点击苏打水按 30ml 或 45ml 单位直接补入杯中 |
| 薄荷 | 8 片 | 点击薄荷按 1 或 2 片加入 |
| 冰块 | 6 块 | 点击加冰，每次 1 块 |
| 摇酒 | 3 次 | 点击摇酒，每次 1 次 |

### 3.2 推荐交互

底部原料区：

- 点击白朗姆/青柠汁/糖浆：将一个 15ml 单位装入 jigger。
- 如果 jigger 中已有其他原料，先提示或禁止切换。
- 点击“量杯入杯”：把当前 15ml 单位倒入成品杯。
- 点击苏打水：直接向成品杯加入固定单位。
- 点击薄荷：加入固定片数。

动作区：

- `量杯入杯`
- `加冰`
- `摇酒`
- `出杯`
- `清空量杯`

不再使用：

- 长按连续倒酒。
- 任意 ml 容量。
- P0.8 Mojito 的雪克杯。

---

## 4. 是否需要雪克杯

P0.8 Mojito 不建议加入雪克杯。

原因：

- Mojito 通常是 build/stir 类流程，不是 shaken cocktail。
- 加雪克杯会引入第三个容器，让当前已经混乱的量杯/成品杯关系更复杂。
- P0.8 的目标是把一杯 Mojito 做清楚，不是覆盖所有调酒技法。

雪克杯应该留给 P1 或第二款酒：

- Daiquiri
- Margarita
- Whiskey Sour

当引入 shaken drink 时，再建立：

```text
jigger -> shaker -> glass
```

Mojito 当前建议保持：

```text
jigger -> glass
```

---

## 5. UI 调整方向

- 量杯和成品杯放大，成为调酒页主视觉。
- 舞台减少空洞留白。
- 底部原料保持可点，但不抢主舞台。
- 原料卡片应显示当前进度，例如 `30/45ml`，但不承担连续倒酒反馈。
- 动作按钮 P0.8 保留 `摇酒`；但舞台不显示雪克杯，避免产生第三容器。

## 6. 实施拆分

### Batch 1：视觉比例

- 放大量杯和成品杯。
- 重新分配舞台空间，减少空洞。
- 保持现有数据模型不变。

当前状态：In Progress。已在 iPhone 17 Simulator 验收杯具放大版，截图：`/tmp/mixcrit-p0.8-larger-vessels.png`。

### Batch 2：离散 jigger 单位

- 移除长按连续倒酒。
- 白朗姆/青柠汁/糖浆点击后装入 15ml jigger。
- `量杯入杯` 每次转移一个 jigger 单位。

当前状态：Next。现有代码仍然是长按连续倒料：

- jigger 原料每 145ms 增加约 1.5ml。
- 苏打水每 145ms 增加约 6ml。
- 薄荷每 145ms 增加 1 片。

### Batch 3：动作语义统一

- P0.8 保留 `摇酒` 动作文案。
- 不在舞台中展示雪克杯。
- 后续若改为真实 Mojito 技法，再将评分中的 `shakeCount` 映射或迁移为 `stirCount`。
