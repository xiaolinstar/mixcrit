## 2. 设计目标

调酒页应让玩家在 3 秒内理解：

1. 我要做的是 Mojito。
2. 左边是量杯，右边是成品杯。
3. 原料点击后会产生明确的一步动作。
4. 白朗姆等标准酒类要通过 15ml 量杯多次加入。
5. P0.8 不显示雪克杯，最后点击 `摇酒` 后出杯。

## 3. 推荐页面原型

### 3.1 竖屏结构

```text
┌──────────────────────────────┐
│ 订单 Mojito        客人偏好   │  高度小，只显示关键信息
│ 当前步骤：加入白朗姆 0/45ml   │
├──────────────────────────────┤
│                              │
│        ┌─────┐      ┌─────┐  │
│        │量杯 │  →   │成品杯│  │
│        │15ml │      │     │  │
│        └─────┘      └─────┘  │
│                              │
│                              │  舞台内不显示状态文字
│                              │
├──────────────────────────────┤
│ 白朗姆  青柠汁  糖浆  苏打水  薄荷 │  原料区
│ 0/45   0/15   0/15  0/90   0/8 │
├──────────────────────────────┤
│ [量杯入杯] [加冰 0/6]        │
│ [摇酒 0/3] [出杯] [清空量杯] │
└──────────────────────────────┘
```

### 3.2 舞台对象

只保留：

- 物理 jigger
- 成品杯
- 左到右弱方向线
- 杯内液体、冰块、薄荷、气泡反馈

不显示：

- 吧台板
- 雪克杯
- `量杯 / 成品杯` 文字标签
- 外层刻度量杯
- 非必要装饰背景

## 4. 推荐玩法原型

### 4.1 核心动作

| 动作 | 输入 | 结果 |
|------|------|------|
| 点击白朗姆 | 原料卡 | 量杯装入白朗姆 15ml |
| 点击量杯入杯 | 动作按钮 | 白朗姆 15ml 倒入成品杯 |
| 重复 3 次 | 原料 + 量杯入杯 | 白朗姆达到 45ml |
| 点击青柠汁 | 原料卡 | 量杯装入青柠汁 15ml |
| 点击糖浆 | 原料卡 | 量杯装入糖浆 15ml |
| 点击苏打水 | 原料卡 | 成品杯增加固定单位苏打水 |
| 点击薄荷 | 原料卡 | 成品杯加入固定片数薄荷 |
| 点击加冰 | 动作按钮 | 成品杯增加 1 块冰 |
| 点击摇酒 | 动作按钮 | 摇酒次数 +1 |
| 点击出杯 | 动作按钮 | 进入评分页 |

### 4.2 固定单位

| 原料 | 单次单位 | 目标 | 需要次数 |
|------|----------|------|----------|
| 白朗姆 | 15ml | 45ml | 3 |
| 青柠汁 | 15ml | 15ml | 1 |
| 糖浆 | 15ml | 15ml | 1 |
| 苏打水 | 30ml | 90ml | 3 |
| 薄荷 | 2 片 | 8 片 | 4 |
| 冰块 | 1 块 | 6 块 | 6 |
| 摇酒 | 1 次 | 3 次 | 3 |

## 5. 状态规则

### 5.1 量杯状态

```text
Empty
  ↓ 点击白朗姆/青柠汁/糖浆
Filled(ingredient, 15ml)
  ↓ 点击量杯入杯
Empty + 成品杯增加 15ml
```

限制：

- 量杯非空时，点击其他 jigger 原料不应直接替换。
- 可以提示玩家先点 `量杯入杯` 或 `清空量杯`。
- 不允许 17ml、22ml 等非标准容量。

### 5.2 直接入杯原料

苏打水、薄荷、冰块直接进入成品杯，不经过 jigger。

## 6.1 量杯视觉决策

当前实现的问题：

- 物理 jigger 贴图之外曾叠加外框、刻度线和目标数字，视觉上像“现实量杯外面套了一层玻璃杯”。
- 这种表达既不真实，也会让玩家误解量杯是一个可连续读数的容器。

P0.8 新方向：

- 舞台只显示一个写实 jigger，不再显示外层刻度量杯。
- jigger 表达的是一个固定 15ml 容量单位，不是连续刻度容器。
- 量杯是否为空、已装什么原料，优先通过 jigger 内部液体颜色和顶部状态栏表达。
- 不在 jigger 旁边显示文字标签。
- 不在 jigger 上显示 `15/30/45/60` 刻度。

仿真方向：

- 金属 jigger 使用真实金属材质，高反射边缘、中心暗部、杯口椭圆高光。
- 装入 15ml 时，jigger 内只出现一个明确的小液面，不出现可任意调节的刻度。
- 成品杯使用写实高球杯素材，避免程序额外绘制上下双圈。

### 9.1 底部数值语义

底部原料卡的数值不是库存上限，而是酒谱进度：

- `已加`：玩家已经加入成品杯的数量。
- `目标`：本杯 Mojito 的推荐/评分目标数量。

示例：

- 白朗姆：`已加 0ml / 目标 45ml`
- 青柠汁：`已加 0ml / 目标 15ml`
- 薄荷：`已加 0片 / 目标 8片`

当前实现已将 `0/45ml` 这类写法改为 `已加 0ml / 目标 45ml`，避免误解为库存上限。

### 9.1.1 P0.8 写实资产替换进展

已完成：

- 首页主视觉替换为写实 `jigger_empty` + `highball_mojito_full`。
- 调酒页量杯替换为写实金属 `jigger_empty`。
- 点击白朗姆/青柠汁/糖浆装入量杯时，有短暂倒入线和量杯脉冲；静态状态保留杯口液面与顶部状态。

暂缓：

- `highball_glass_empty` 暂未替换。当前 imagegen 的透明玻璃素材带明显 chroma-key 背景折射，直接接入会产生洋红色玻璃边缘，不符合参考图目标。

### 9.2 目标视觉

P0.8 的目标效果应接近参考图：

- 顶部信息条：紧凑、横向、突出当前订单和当前原料进度。
- 主舞台：量杯和成品杯占据核心视觉空间，比例大，背景干净。
- 量杯：写实金属 jigger，杯口有真实液面，不显示外层刻度杯。
- 成品杯：写实高球杯，包含液体折射、冰块、薄荷、青柠、气泡。
- 底部原料：写实瓶/杯/薄荷素材，原料大于动作按钮。
- 动作按钮：小而清楚，位于底部安全区上方。

### 9.3 当前差距

当前实现的布局与玩法已经接近目标，但真实感仍有明显差距：

- 当前成品杯是空杯贴图 + 程序绘制液体/冰/薄荷，无法达到参考图的玻璃折射和冰块层次。
- 当前原料瓶是独立素材，但光影、透视、材质风格还没有完全统一。
- 当前舞台背景仍是程序绘制的暗面板，不是参考图中带摄影质感的吧台暗背景。

因此，若要达到参考图的真实感，需要进入“写实资产替换”阶段，而不是继续只调 SwiftUI/SpriteKit 形状。
