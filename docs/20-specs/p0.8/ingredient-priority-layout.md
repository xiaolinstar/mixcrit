# Spec: P0.8 原料与动作按钮主次重排

> 创建日期：2026-06-14  
> 阶段：P0.8  
> 目标版本：0.0.9  
> 状态：In Progress

---

## 1. 背景

当前调酒页底部区域中，原料卡片偏小，动作按钮偏大。对于调酒游戏来说，玩家的主要操作对象应该是“原料”和“杯具”，按钮只是工具动作。当前主次关系反了：

- 原料是玩家调酒的核心，但视觉权重不足。
- “量杯入杯 / 加冰 / 摇酒 / 出杯”按钮过大，像主玩法入口。
- 原料区和调酒区边界曾经混在一起，玩家难以判断原料属于舞台还是控制区。

---

## 2. 目标

- 原料展示更大、更清楚，成为底部控制区第一主角。
- 动作按钮缩小，但保持可点击和可读。
- 原料区贴近底部，但不侵入 SpriteKit 调酒舞台。
- 小屏设备仍然单屏完整显示。

---

## 3. 非目标

- 不在本 Spec 中新增原料。
- 不改变配方和评分规则。
- 不引入拖拽原料到杯中的新交互。
- 不重做美术资产。

---

## 4. 设计方向

建议底部结构：

```text
Ingredient Shelf   高度较大，展示 5 个原料
Primary Actions    两行紧凑按钮
Secondary Actions  更小、更弱的清空/重置
```

初步比例建议：

| 区域 | 当前问题 | 建议 |
|------|----------|------|
| 原料区 | 太小，像状态标签 | 高度增加 15%~25%，图标/瓶身更大 |
| 主动作按钮 | 过大，抢原料权重 | 高度减少 10%~18% |
| 次动作按钮 | 可见即可 | 更弱、更低、更小 |
| Dock 背景 | 已变为不透明 | 继续保持，避免舞台透出 |

---

## 5. 验收标准

- [ ] iPhone 17e 上原料瓶/薄荷比当前版本更醒目。
- [ ] 动作按钮仍能清楚点击，但视觉权重低于原料区。
- [ ] 原料区不与 SpriteKit 舞台边界重叠。
- [ ] iPhone SE 尺寸上不出现按钮文字截断或关键内容出屏。
- [ ] 用户能判断：原料用于选择/倒料，按钮用于工具动作。
