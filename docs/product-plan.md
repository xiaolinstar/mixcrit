# 调酒游戏产品规划文档

调研日期：2026-05-19  
项目：mixcrit  
产品方向：真实调酒模拟 + 轻经营养成  
目标平台：iOS

## 1. 产品定位

本产品不是单纯的鸡尾酒配方工具，也不是普通换皮经营小游戏，而是一款结合真实调酒学习、调酒操作模拟、库存管理和酒吧经营养成的 iOS 游戏。

一句话定位：

> 玩家经营一家小酒吧，通过真实的调酒操作服务客人，学习配方、管理库存、升级酒吧，并逐步成为顶级调酒师。

核心价值：

- 让用户通过游戏学习真实鸡尾酒调制方法
- 通过高质量视觉、动画和触感反馈，让调酒过程具备沉浸感
- 通过库存、进货、收益和客人满意度，形成经营养成循环
- 通过配方、营养、热量和酒精度计算，提供一定现实参考价值

## 2. 产品方向选择

当前有两个方向：

1. 调酒模拟器
   - 学习调酒
   - 调酒评分
   - 良好视觉呈现
   - 根据已有消耗品推荐可调酒品
   - 酒品、辅料、库存管理
   - 根据调酒自动扣减库存
   - 计算酒精度、热量、糖分等信息

2. 酒吧经营养成游戏
   - 经营一家酒铺或酒吧
   - 客人点酒
   - 玩家调酒
   - 进货、升级、收益管理
   - 调酒过程可作为小游戏
   - 支持同时调多杯、限时调酒等挑战

建议方案：

> 底层先做真实调酒模拟器，上层用酒吧经营养成包装。

原因：

- 调酒模拟是产品灵魂，决定差异化和口碑
- 经营养成提供长期目标，决定留存和商业化
- 库存和进货可以同时服务“真实工具”和“经营游戏”
- 视觉和动画资产可以在学习、经营、挑战等模式中复用

## 3. 目标用户

### 3.1 核心用户

- 喜欢鸡尾酒的新手
- 对调酒感兴趣但缺少线下练习环境的人
- 喜欢模拟经营游戏的玩家
- 喜欢精致食物、饮品、咖啡、烘焙类游戏的用户

### 3.2 次级用户

- 家庭调酒爱好者
- 想管理家中酒水和辅料库存的人
- 对酒精度、热量和糖分敏感的用户
- 想了解鸡尾酒文化和配方背景的人

## 4. 核心体验原则

1. 调酒过程必须好看
   - 液体流动、冰块碰撞、杯壁冷凝、摇壶震动、装饰落杯都应有明确表现。

2. 操作必须有反馈
   - 倒多了、摇少了、冰加错了、步骤反了，都应被系统识别并反馈。

3. 结果必须可理解
   - 分数不只是数字，还要说明为什么这杯酒好或不好。

4. 游戏目标必须持续存在
   - 客人、订单、收益、库存、升级和解锁，让用户愿意继续玩。

5. 真实知识要自然嵌入
   - 不用大段讲课，而是在评分、配方卡和关卡结算中给出关键知识。

## 5. 产品结构

产品分为三层：

### 5.1 调酒模拟层

负责核心手感和视觉表现。

功能包括：

- 倒酒
- 加冰
- 摇酒
- 搅拌
- 过滤
- 装饰
- 出杯
- 调酒评分
- 风味和酒精度计算

### 5.2 工具与知识层

负责现实参考价值。

功能包括：

- 配方学习
- 材料说明
- 酒品库存
- 辅料库存
- 根据已有材料推荐可调酒品
- 热量、糖分、酒精度计算
- 调酒知识卡片

### 5.3 经营养成层

负责长期留存。

功能包括：

- 酒吧主界面
- 客人点单
- 每日营业
- 订单队列
- 收入和小费
- 进货
- 酒吧升级
- 解锁新配方、新杯具、新装饰和新客群

## 6. 版本路线

### 6.1 P0 原型版本

目标：验证“手机上调一杯酒”是否好玩。

范围：

- 一个调酒台场景
- 3 款鸡尾酒
- 倒酒、加冰、摇酒、出杯
- 简单评分
- 基础液体和冰块动画

验收标准：

- 用户能在 3 分钟内完成第一杯酒
- 用户能理解自己为什么得分高或低
- 调酒过程有明显视觉反馈

### 6.2 P1 MVP 版本

目标：形成可上架的基础经营游戏闭环。

范围：

- 小酒吧经营主循环
- 10 款经典鸡尾酒
- 客人点单
- 调酒小游戏
- 评分系统
- 库存自动扣减
- 进货系统
- 收入、小费和基础升级
- 配方学习卡
- 热量、糖分、酒精度基础计算

验收标准：

- 用户可以完成一个完整营业日
- 调酒、评分、收钱、扣库存、进货形成闭环
- 至少 10 款酒的视觉和数据完整
- 动画表现足以支撑上架展示图和短视频素材

### 6.3 P2 增强版本

目标：提升留存和内容深度。

范围：

- 自由调酒模式
- 根据已有库存推荐可调酒品
- 每日挑战
- 更多客人类型
- 高级风味分析
- 酒吧装修
- 更多配方和材料

### 6.4 P3 长线版本

目标：形成长期运营能力。

范围：

- 员工系统
- 自创酒品上架
- 季节活动
- 比赛模式
- 多个酒吧地点
- AI 推荐配方
- 社区分享

## 7. MVP 核心循环

MVP 主循环：

1. 进入酒吧
2. 开始营业
3. 客人进店并点酒
4. 玩家接受订单
5. 进入调酒台
6. 根据配方选择材料和工具
7. 完成调酒操作
8. 系统评分
9. 客人反馈
10. 获得收入、小费和经验
11. 库存自动扣减
12. 营业结束后查看报表
13. 进货或升级
14. 解锁新内容

## 8. MVP 子流程细化

### 8.1 首次进入流程

目标：让用户快速理解游戏。

流程：

1. 展示年龄确认和责任饮酒提示
2. 展示酒吧开场画面
3. 进入新手教学
4. 第一个客人点一杯 Mojito
5. 系统引导玩家完成倒酒、加冰、摇酒或搅拌、装饰、出杯
6. 给出评分
7. 解锁酒吧主界面

关键设计：

- 第一次不要展示复杂菜单
- 教学应结合真实操作，不做长篇说明
- 每一步都要有动画和触感反馈

### 8.2 酒吧主界面流程

目标：承载经营状态和下一步行动。

主界面元素：

- 酒吧背景
- 当前日期 / 营业日
- 金币 / 现金
- 店铺评分
- 库存提醒
- 开始营业按钮
- 进货入口
- 配方入口
- 升级入口

交互流程：

1. 用户进入酒吧主界面
2. 查看库存是否不足
3. 可选择进货或直接开始营业
4. 点击开始营业进入订单场景

动画重点：

- 酒吧灯光开关
- 吧台环境微动效
- 客人剪影或门口动效
- 现金和评分变化时的轻量反馈

### 8.3 客人点单流程

目标：制造经营压力和任务目标。

流程：

1. 客人进入
2. 系统生成订单
3. 展示酒名、等待时间、客人偏好
4. 玩家接受订单
5. 进入调酒台

订单字段：

- 酒品名称
- 难度
- 推荐售价
- 客人耐心值
- 特殊要求
- 可获得小费范围

示例：

- 客人点单：Whiskey Sour
- 特殊要求：不要太酸
- 等待时间：90 秒
- 小费倾向：出杯快且平衡度高时增加

动画重点：

- 客人入场
- 表情变化
- 点单气泡
- 耐心条下降

### 8.4 调酒台流程

目标：完成核心操作体验。

调酒台区域：

- 中央：杯具 / 摇壶 / 混合容器
- 左侧：酒瓶和利口酒
- 右侧：辅料、果汁、糖浆、装饰
- 底部：工具栏
- 顶部：订单、时间、评分预览

基础流程：

1. 选择杯具或摇壶
2. 加冰
3. 倒入材料
4. 执行摇酒或搅拌
5. 过滤入杯
6. 添加装饰
7. 点击出杯

交互方式：

- 拖动酒瓶到容器上方
- 长按或倾斜倒酒
- 毫升数实时变化
- 拖动冰块入杯
- 摇动手机或按节奏摇酒
- 旋转手势搅拌
- 滑动完成过滤
- 拖拽装饰物到杯口

动画重点：

- 液体从瓶口流出
- 不同颜色液体混合
- 冰块落杯并碰撞
- 摇壶晃动
- 液体起泡或变浑浊
- 杯壁冷凝
- 装饰物落位

### 8.5 倒酒子流程

目标：让“毫升控制”成为有趣且可学习的操作。

流程：

1. 玩家选择一瓶酒
2. 拖到目标容器
3. 按住开始倒酒
4. 系统显示实时毫升数
5. 松手停止
6. 判断误差

评分因素：

- 目标用量
- 实际用量
- 误差范围
- 是否倒入正确容器
- 是否倒错材料

动画设计：

- 瓶身倾斜
- 酒液连续流动
- 流速根据倾斜角变化
- 液面逐渐上升
- 倒酒结束有一滴收尾

图片资产：

- 不同瓶型
- 酒标占位图
- 酒液颜色贴图
- 量杯或容器刻度

### 8.6 加冰子流程

目标：表达温度和稀释度。

流程：

1. 玩家选择冰块
2. 拖动到杯或摇壶
3. 冰块落入容器
4. 系统记录冰量
5. 影响温度和稀释速度

评分因素：

- 冰块数量
- 加冰时机
- 是否符合配方要求

动画设计：

- 冰块落下
- 轻微弹跳
- 与杯壁碰撞
- 液体飞溅
- 冰块逐渐透明或变小

图片资产：

- 方冰
- 碎冰
- 大冰球
- 冰铲
- 冰桶

### 8.7 摇酒子流程

目标：让摇酒成为核心爽点。

流程：

1. 玩家将材料和冰加入摇壶
2. 盖上摇壶
3. 摇动手机或进行屏幕节奏操作
4. 系统统计摇酒时长、节奏和强度
5. 影响温度、稀释度、泡沫和混合度

评分因素：

- 摇酒时长
- 摇酒强度
- 节奏稳定性
- 是否需要摇酒
- 是否过度摇酒

动画设计：

- 摇壶快速晃动
- 金属高光变化
- 冰块撞击声和震动
- 摇完后出现冷凝和水珠
- 打开摇壶时出现轻微雾气

图片资产：

- 摇壶主体
- 摇壶盖
- 金属高光贴图
- 冷凝水珠效果

### 8.8 搅拌子流程

目标：区分“摇”和“搅”的专业差异。

流程：

1. 玩家选择吧勺
2. 在混合杯中旋转手势
3. 系统统计圈数、速度和持续时间
4. 影响温度、稀释度和清澈度

评分因素：

- 搅拌时长
- 速度稳定性
- 是否保持酒液清澈
- 是否适用于该酒款

动画设计：

- 吧勺旋转
- 冰块绕圈
- 液面形成轻微旋涡
- 玻璃杯折射变化

图片资产：

- 吧勺
- 混合杯
- 透明液体高光
- 旋涡效果

### 8.9 过滤出杯流程

目标：让完成动作有仪式感。

流程：

1. 玩家选择过滤器
2. 将摇壶或混合杯移动到目标杯具上方
3. 滑动或长按完成过滤
4. 酒液进入杯中
5. 系统检查杯具、容量和成品状态

评分因素：

- 是否使用正确杯具
- 是否过滤冰块或果渣
- 出杯容量是否合适
- 是否洒出

动画设计：

- 细流入杯
- 冰块被过滤器挡住
- 液面上升
- 结束时酒液轻微晃动

图片资产：

- 滤冰器
- 不同杯具
- 成品酒液颜色
- 杯中泡沫或分层效果

### 8.10 装饰流程

目标：强化成品展示和评分情绪。

流程：

1. 玩家选择装饰物
2. 拖到杯口或杯中
3. 系统判断位置和类型
4. 成品展示

装饰类型：

- 柠檬片
- 青柠角
- 橙皮
- 樱桃
- 薄荷
- 盐边
- 糖边
- 迷迭香

评分因素：

- 装饰是否正确
- 位置是否准确
- 是否符合客人特殊要求

动画设计：

- 果片落位
- 薄荷轻微摆动
- 橙皮挤油喷雾
- 盐边或糖边附着

图片资产：

- 各类装饰物
- 杯口挂饰状态
- 喷雾或香气效果

### 8.11 评分结算流程

目标：让玩家知道哪里做得好，哪里要改。

评分维度：

- 配方准确度
- 用量准确度
- 步骤顺序
- 操作速度
- 温度
- 稀释度
- 风味平衡
- 装饰完成度
- 客人偏好匹配度

结算展示：

- 总评分：S / A / B / C / D
- 收入
- 小费
- 经验
- 库存消耗
- 关键改进建议

示例反馈：

- 威士忌多倒了 6ml，酒感偏强
- 摇酒时间偏短，温度不够低
- 柠檬汁略多，酸度偏高
- 装饰正确，客人满意度提升

动画重点：

- 成品酒特写
- 客人表情反馈
- 分数逐项跳出
- 金币和经验增长

## 9. MVP 数据设计

### 9.1 鸡尾酒数据

字段建议：

- id
- 名称
- 分类
- 难度
- 杯具
- 所需工具
- 材料列表
- 标准步骤
- 推荐摇酒或搅拌时间
- 目标风味
- 默认售价
- 热量估算
- 酒精度估算
- 解锁条件

### 9.2 材料数据

字段建议：

- id
- 名称
- 类型
- 单位
- 当前库存
- 单位成本
- 每单位热量
- 每单位糖分
- 酒精度
- 是否可消耗
- 保质期

材料类型：

- 基酒
- 利口酒
- 果汁
- 糖浆
- 碳酸饮料
- 香草
- 水果
- 装饰
- 冰块

### 9.3 库存扣减

示例：

- Gin Tonic
  - 金酒 -45ml
  - 汤力水 -120ml
  - 青柠 -0.25 个
  - 冰块 -5 颗

- Mojito
  - 白朗姆 -45ml
  - 青柠 -0.5 个
  - 薄荷 -8 片
  - 糖浆 -15ml
  - 苏打水 -90ml
  - 冰块 -8 颗

### 9.4 营养和酒精计算

MVP 先支持：

- 总容量
- 酒精度 ABV
- 标准杯数量
- 热量
- 糖分

暂不建议第一版做过细营养表，例如蛋白质、脂肪、维生素等。维护成本较高，且与核心游戏循环关系较弱。

## 10. MVP 推荐酒单

第一版建议 10 款经典鸡尾酒：

1. Mojito
2. Gin Tonic
3. Whiskey Sour
4. Old Fashioned
5. Margarita
6. Martini
7. Negroni
8. Daiquiri
9. Cuba Libre
10. Cosmopolitan

选择理由：

- 覆盖摇酒、搅拌、直调等不同技法
- 材料复用率高
- 适合做难度递进
- 用户认知度高
- 视觉表现差异明显

## 11. 动画设计规划

动画是本产品成败关键之一。它不仅是装饰，而是操作反馈、专业感和爽感的来源。

### 11.1 动画优先级

P0 必做：

- 倒酒流体
- 液面上升
- 冰块落杯
- 摇壶晃动
- 成品酒展示

P1 必做：

- 不同液体颜色混合
- 杯壁冷凝
- 客人表情变化
- 金币和经验结算
- 装饰物落位

P2 可做：

- 更复杂液体分层
- 气泡上升
- 果汁浑浊扩散
- 橙皮挤油喷雾
- 客人入场和离场动画

### 11.2 动画设计原则

- 每个关键操作都要有即时反馈
- 动画时长要短，不能拖慢游戏节奏
- 成品展示可以更有仪式感
- 错误操作也要有视觉反馈
- 液体、冰块、玻璃和金属是视觉重点

### 11.3 推荐实现方式

原生 iOS 推荐：

- SwiftUI：界面状态、转场、结算动画
- SpriteKit：调酒台、液体、冰块、工具交互
- Core Haptics：倒酒、冰块碰撞、摇酒、评分反馈
- Core Motion：摇动手机、倾斜倒酒

## 13. MVP 开发拆分建议

### 13.1 第一阶段：调酒台原型

功能：

- 单个调酒台

重点：

## 14. 风险与控制

### 14.1 范围风险

风险：模拟、经营、库存、营养、动画、图片都很重要，容易范围过大。

控制方式：

- P0 只验证调酒手感
- P1 只做轻经营
- 库存先做基础扣减
- 营养先做热量、糖分、ABV

### 14.2 资产成本风险

风险：每款酒、杯具、材料、客人都需要图片和动画，成本高。

控制方式：

- 优先选择材料复用率高的酒单
- 成品酒通过颜色、装饰和杯型组合生成
- 客人数量先少，但表情反馈要清晰
- 动画先做关键动作，不追求全物理真实

### 14.3 专业准确性风险

风险：真实调酒用户会关注配方和技法准确性。

控制方式：

- MVP 只选经典配方
- 配方数据来源保持一致
- 评分逻辑允许一定误差
- 明确这是模拟训练，不替代专业课程

## 15. 结论

建议将产品规划为“真实调酒模拟 + 轻经营养成”的混合型 iOS 游戏。

第一阶段不要追求大而全，而是把三个点做好：

1. 调酒操作真的有趣
2. 成品酒真的好看
3. 经营循环真的能跑起来

当这三个点成立后，再逐步扩展库存推荐、自由调酒、热量分析、酒吧装修、员工系统和社区分享。

MVP 的关键目标是：

> 用户在一个小酒吧里服务客人，完成 10 款经典鸡尾酒的调制，并通过视觉、动画、评分和经营反馈，产生“我还想再开一天店”的感觉。
