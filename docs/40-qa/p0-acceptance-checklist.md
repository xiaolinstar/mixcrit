# P0 Mojito 原型验收清单

> 适用版本：`0.0.12 (12)` / P0.10  
> 验收日期：2026-06-18  
> 目标：确认「可理解、可完成、可反馈」的核心调酒体验已经稳定

---

## 1. 核心流程验收（必须通过）

- [x] 从酒吧首页点击「开始营业」可进入调酒台
- [x] 订单卡片能显示 Mojito 与当前步骤提示
- [x] 可选择 5 种原料：白朗姆、青柠汁、糖浆、苏打水、薄荷
- [x] 可执行核心动作：点击装量杯/加料、量杯入杯、加冰、摇酒、出杯
- [x] 点击出杯后进入评分页，显示分数、评级、反馈条目
- [x] 可从评分页执行「再调一杯」和「回到酒吧」

---

## 2. 交互边界验收（P0 收口重点）

- [x] 点击原料装入量杯或直接入杯后，动画与数值能正确更新
- [x] 快速连续点击原料或动作按钮，不会出现状态异常
- [x] 量杯有剩余液体时点击出杯，液体会自动计入本杯后再评分
- [x] 返回酒吧时不会出现「先跳回调酒台再切回首页」的闪动
- [x] 连续快速点击加冰/摇酒/出杯，不会导致页面卡死或状态异常

### 标准配方操作步骤（点击式 15ml 量杯）

| 原料 | 目标 | 操作 |
|------|------|------|
| 白朗姆 | 45ml | 点击装 15ml → 量杯入杯，**重复 3 次** |
| 青柠汁 | 15ml | 点击装 15ml → 量杯入杯，**1 次** |
| 糖浆 | 15ml | 点击装 15ml → 量杯入杯，**1 次** |
| 苏打水 | 90ml | 直接点击，每次 +30ml，**3 次** |
| 薄荷 | 8 片 | 直接点击，每次 +2 片，**4 次** |
| 冰块 | 6 颗 | 点「加冰」6 次 |
| 摇酒 | 3 次 | 点「摇酒」3 次 |

---

## 3. 配方与评分验收（建议基准）

### 推荐基准配方（目标）
- 白朗姆：45ml
- 青柠汁：15ml
- 糖浆：15ml
- 苏打水：90ml
- 薄荷：8 片
- 冰块：约 6 块
- 摇酒：至少 3 次

### 评分反馈检查
- [x] 缺少某原料时，会出现对应「缺少 xxx」反馈
- [x] 冰块过少/过多时，会出现温度或稀释相关反馈
- [x] 摇酒不足时，会出现融合度不足反馈
- [x] 接近目标用量时，能看到正向反馈

---

## 4. 演示通过标准（最小可发布到内部测试）

- [x] 新用户在无讲解情况下可完成一杯 Mojito（P0.10 引导 + 动态提示）
- [x] 演示 3 轮（标准配方 / 明显偏差 / 随机操作）均可得到合理反馈
- [x] 无阻断性问题（崩溃、无法出杯、无法重置、页面卡死）

## 6. 遗留项（不阻塞 0.0.12 发布）

- 主舞台上方暗区偏大 → 归入 P0.11
- 倒入/加冰/摇酒动态反馈偏弱 → 归入 P0.11
